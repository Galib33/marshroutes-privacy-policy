01 Information We Collect

a) Location Data

The Application requests location permission (precise and/or approximate) to provide map, route, and location-based functionality. If location access is denied, some features may not work.

b) Usage and Diagnostic Data

We use Firebase Analytics and Firebase Performance Monitoring to understand app usage and improve reliability, speed, and user experience. This may include app interaction events, device/app metadata, and performance diagnostics.

c) Push Notification Data

If you grant notification permission, we use Firebase Cloud Messaging (FCM) to send push notifications. FCM may generate a device token used for notification delivery.

d) Local Device Storage

The Application stores certain data locally on your device (such as saved routes/markers, app preferences, and cached values) using local storage technologies, including SharedPreferences, secure storage, and SQLite.
